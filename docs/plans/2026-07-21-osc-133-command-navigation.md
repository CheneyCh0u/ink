# OSC 133 命令跳转与复制实施计划

关联 Issue：#58

## 目标

基于已有 OSC 133 标记，为活动终端提供上一条/下一条命令跳转、复制命令、复制输出；缺少或不完整标记时静默无操作。

## 约束与设计

- `TerminalCore` 保持纯 Swift，不依赖 AppKit/Metal。
- `Cell` 保持 8 字节，`RowInfo` 保持 2 字节。
- `RowInfo` 在现有 16 位内编码软折行、语义类型及可选转换列；同行多转换只把
  被覆盖的必要点放入终端级稀疏旁路表，不增加普通行的常驻内存。
- 命令块只在用户触发动作时扫描 `scrollback + grid`，不维护重复文本或每格索引。
- reflow 将语义转换列映射到新的物理行；软折行复制时不插入换行，硬换行保留。

## 步骤

1. 为 `RowInfo` 的转换列编码、OSC 133 落点及 reflow 映射补失败测试。
2. 在 `TerminalCore` 增加命令块扫描、相邻命令定位、命令/输出文本提取 API，并覆盖跨 scrollback、软折行、缺失标记。
3. 在 `TerminalMetalView` 增加跳转和写剪贴板动作，复用现有滚动定位。
4. 将动作经工作区路由到活动 pane，在“编辑”菜单加入原生命令与快捷键。
5. 运行全量测试与构建，审查 diff 后提交、推送并创建 `Closes #58` 的 PR。
