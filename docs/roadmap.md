# 功能清单与路线图

本文件是功能范围的**唯一权威来源**。实际执行进度用 todolist 跟踪，两者配套：本文件定义"做什么、为什么"，todolist 记录"做到哪了"。

范围有变动时先改本文件，再调整 todolist。

## 优先级定义

- **P0** — 不做就不算能用的终端，M1–M4 内完成
- **P1** — 决定好不好用，M5–M6 及之后
- **不做** — 明确排除，见文末。想加回来需要先讨论

---

## P0：核心

### PTY 与进程

- `forkpty` 启动登录 shell，`argv[0]` 带前导 `-`（否则不读 `.zprofile`）
- 环境变量：`TERM=xterm-256color`、`COLORTERM=truecolor`、`LANG`、`TERM_PROGRAM=ink`
- 窗口尺寸同步：`TIOCSWINSZ` + `SIGWINCH`，resize 后 vim 立即重排
- 子进程退出处理、僵尸回收
- 非阻塞 I/O + 背压：`yes` 这类高速输出不得阻塞 UI 线程

### VT 解析

- UTF-8 增量解码，跨 read 边界的半个字符不能丢
- CSI / OSC / DCS / ESC 状态机
- SGR：粗体、斜体、下划线、反显、删除线、隐藏；16 色 / 256 色 / 真彩色
- 光标：移动、保存恢复、形状、显隐
- 擦除与编辑：清屏清行、插入删除行列
- 滚动区域 `DECSTBM`
- **备用屏（alternate screen）** — vim / less / htop 依赖
- DEC 私有模式开关

验收标准：`vttest` 核心项通过。

### 字符宽度

- `wcwidth` 语义，东亚全角占两列
- 组合字符（变音符号）
- emoji ZWJ 序列、变体选择符

> 这块出错的表现是中文不对齐、光标漂移。是中文用户最先感知到的质量问题，不能含糊。

### 渲染

- Metal glyph atlas + 实例化绘制，每帧单次 draw call
- 粗体 / 斜体使用真字重，不用倾斜变换伪造
- 彩色 emoji，独立 RGBA atlas
- 光标绘制：块 / 竖线 / 下划线，可闪烁
- 高 DPI，多显示器不同缩放率切换时正确重建 atlas

### 输入

- `NSTextInputClient` 完整实现：中文输入、候选窗跟随光标、预编辑下划线
- 修饰键组合、功能键转义序列
- Option 键行为可选（Meta vs 重音输入）
- **Bracketed paste** — 粘贴多行不被当命令执行，属安全需求
- 鼠标上报（SGR 协议）— vim / tmux 依赖

### 交互

- scrollback + 平滑滚动
- 鼠标选中、双击选词、三击选行、Option 块选
- 复制粘贴、选中即复制（可配）
- resize 时的重排 —— **P0 阶段历史行截断，不做 reflow**

### Shell 集成（OSC 133）— 提前到 P0

语义化 prompt 标记：区分提示符、命令、输出。

> **为什么提前：** 它要求 `TerminalCore` 的行数据带上语义标记，后补需要改动数据结构和 scrollback 压缩格式，代价远高于一开始就留位。P0 只做「解析并记录标记」，基于它的功能（跳转命令、复制输出、耗时显示）放 P1。

## P1：体验

### Reflow

窗口变窄时历史长行重新折行。

> **与内存压缩方案耦合，必须一起设计。** 压缩存储若假设行长固定就无法 reflow。P0 阶段虽然不实现，但 grid 数据结构要预留可行性。

### 会话与窗口

- 标签页
- 侧边栏三态：完整项目列表 / 项目图标轨道 / 完全隐藏
- 项目颜色标记：Finder 式标签色，图标轨道中保留颜色识别
- 会话恢复：重启后回到原目录与布局
- 分屏（水平 / 垂直）—— **优先级最低，可砍**。牵动渲染与焦点管理，若日常用 tmux 则无必要

### 基于 OSC 133 的功能

- 跳转到上 / 下一个命令
- 只复制命令输出
- 命令执行时长显示
- 失败命令高亮

### 其它

- scrollback 搜索：增量、高亮
- URL 自动识别 + 点击打开
- OSC 8 超链接
- 配置文件（TOML）+ 热重载
- 主窗口内嵌设置中心：外观、启动侧边栏、窗口、字体、光标、交互与 scrollback
- iCloud 配置同步：本机设置变化后自动上传，云端配置由用户手动拉取
- 主题配色、字体配置
- 默认跟随 macOS 浅色 / 深色外观，系统外观切换时实时更新
- 快捷键自定义
- OSC 52 剪贴板（SSH 中复制回本地）
- 通知（OSC 9 / 777）
- 帧率跟随显示器刷新率，失焦降频

## 明确不做

| 功能 | 理由 |
|---|---|
| Sixel / Kitty 图片协议 | 实现量大，与"内存小"定位冲突，受众窄 |
| tmux control mode | 集成复杂度极高，iTerm2 投入多年 |
| 内置 SSH / 端口转发 | 超出终端职责 |
| 插件系统 / 脚本引擎 | 引入运行时即引入常驻内存开销 |
| Linux / Windows 移植 | 与 Swift 选型冲突，要做需推翻技术栈 |
| AI 集成 | 除非确立为产品差异点，否则是范围蔓延 |

## 里程碑

| | 目标 | 内容 |
|---|---|---|
| **M1** | 能跑 | PTY 打通，`NSTextView` 占位显示输出，验证 shell 交互 |
| **M2** | 自绘 | Metal 渲染管线 + glyph atlas 替换占位视图 |
| **M3** | 正确 | VT 兼容性、字符宽度、scrollback、选中复制、OSC 133 标记 |
| **M4** | 输入法 | `NSTextInputClient` 完整实现，中文输入可用 |
| **M5** | 外壳 | 侧边栏、标签页、配置文件 |
| **M6** | 达标 | 内存与帧率实测，scrollback 压缩落地，reflow |

M1 先用占位视图而非直接上 Metal，是为了早期就有可运行、可感知的东西，避免长时间陷在渲染管线里。

指标验收（见 [tech-stack.md](tech-stack.md)）：空闲单窗口 < 50MB，10 万行 scrollback < 200MB，全屏重绘稳定 120fps。
