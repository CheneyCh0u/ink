# 功能清单与路线图

本文件是功能范围的**唯一权威来源**。实际执行进度用 todolist 跟踪，两者配套：本文件定义"做什么、为什么"，todolist 记录"做到哪了"。

范围有变动时先改本文件，再调整 todolist。

## 优先级定义

- **P0** — 不做就不算能用的终端，M1–M4 内完成
- **P1-S** — 下一阶段最先做，直接影响安全、高频操作或产品差异点
- **P1-A** — P1-S 稳定后做，改善多项目与长时间使用体验
- **P1-B** — 有价值但不阻塞主流程，按真实使用反馈排期
- **暂缓** — 当前不进入实现队列，出现明确需求证据后再评估
- **不做** — 明确排除，见文末。想加回来需要先讨论

## 当前基线（2026-07-21）

代码已经越过功能骨架阶段，以下能力已经形成可测试的实现基线：

- P0 核心终端：PTY、VT 解析、字符宽度、Metal 渲染、输入法、选区与 scrollback
- reflow、增量 scrollback 搜索与搜索高亮
- 项目侧边栏、标签页、递归水平 / 垂直分屏与 pane 焦点管理
- TOML 配置与热重载、内嵌设置、主题与字体配置、iCloud 配置同步
- OSC 133 解析、列级语义边界、命令跳转与命令 / 输出复制

当前仍需完成 `vttest` 核心项、真实 120 Hz Instruments 数据和最终内存指标验收。
路线图继续定义范围与顺序，具体完成状态仍以 todolist 为准。

## P0：核心

### PTY 与进程

- `forkpty` 启动登录 shell，`argv[0]` 带前导 `-`（否则不读 `.zprofile`）
- 环境变量：`TERM=xterm-256color`、`COLORTERM=truecolor`、`LANG`、`TERM_PROGRAM=ink`
- 窗口尺寸同步：`TIOCSWINSZ` + `SIGWINCH`，resize 后 vim 立即重排
- 子进程退出处理、僵尸回收
- 非阻塞 I/O + 背压：`yes` 这类高速输出不得阻塞 UI 线程

### VT 解析

- UTF-8 增量解码，跨 read 边界的半个字符不能丢
- CSI / OSC / DCS / ESC 状态机
- SGR：粗体、斜体、下划线、反显、删除线、隐藏；16 色 / 256 色 / 真彩色
- 光标：移动、保存恢复、形状、显隐
- 擦除与编辑：清屏清行、插入删除行列
- 滚动区域 `DECSTBM`
- **备用屏（alternate screen）** — vim / less / htop 依赖
- DEC 私有模式开关

验收标准：`vttest` 核心项通过。

### 字符宽度

- `wcwidth` 语义，东亚全角占两列
- 组合字符（变音符号）
- emoji ZWJ 序列、变体选择符

> 这块出错的表现是中文不对齐、光标漂移。是中文用户最先感知到的质量问题，不能含糊。

### 渲染

- Metal glyph atlas + 实例化绘制，每帧单次 draw call
- 粗体 / 斜体使用真字重，不用倾斜变换伪造
- 彩色 emoji，独立 RGBA atlas
- 光标绘制：块 / 竖线 / 下划线，可闪烁
- 高 DPI，多显示器不同缩放率切换时正确重建 atlas

### 输入

- `NSTextInputClient` 完整实现：中文输入、候选窗跟随光标、预编辑下划线
- 修饰键组合、功能键转义序列
- Option 键行为可选（Meta vs 重音输入）
- **Bracketed paste** — 粘贴多行不被当命令执行，属安全需求
- 鼠标上报（SGR 协议）— vim / tmux 依赖

### 交互

- scrollback + 平滑滚动
- 鼠标选中、双击选词、三击选行、Option 块选
- 复制粘贴、选中即复制（可配）
- resize 时的重排 —— **P0 阶段历史行截断，不做 reflow**

### Shell 集成（OSC 133）— 提前到 P0

语义化 prompt 标记：区分提示符、命令、输出。

> **为什么提前：** 它要求 `TerminalCore` 的行数据带上语义标记，后补需要改动数据结构和 scrollback 压缩格式，代价远高于一开始就留位。P0 只做「解析并记录标记」，基于它的功能（跳转命令、复制输出、耗时显示）放 P1。

## P1：体验

### Reflow

窗口变窄时历史长行重新折行。

> **与内存压缩方案耦合，必须一起设计。** 当前实现已经使用逻辑行语义完成
> reflow；后续修改 scrollback 格式时必须继续验证 10 万行内存和重排性能。

### 会话与窗口

- 标签页
- 侧边栏三态：完整项目列表 / 项目图标轨道 / 完全隐藏
- 项目颜色标记：Finder 式标签色，图标轨道中保留颜色识别
- 分屏：递归水平 / 垂直布局，分屏继承活动 pane 的工作目录
- 会话恢复：重启后恢复项目、标签名、工作目录、分屏结构与比例，再启动新的 shell

> 会话恢复只保存布局意图，不复活 PTY，不持久化终端 scrollback。这样既不制造
> “进程仍在运行”的错觉，也避免增加启动时间、磁盘占用和敏感输出泄露风险。

### P1-S：语义化命令块

- 跳转到上 / 下一个命令
- 只复制命令
- 只复制命令输出
- 记录命令执行时长与退出状态
- 后台命令完成、失败或 Bell 时，在标签上显示安静的状态点

命令操作默认通过菜单和快捷键提供；鼠标靠近命令首行时才显示轻量入口，不能常驻
工具栏或永久占用终端列宽。失败状态不能只靠颜色表达。

### P1-S：安全粘贴

- 保留 bracketed paste 包裹和结束标记过滤
- 粘贴内容包含多行或控制字符时提供确认；目标未开启 bracketed paste 时明确提示风险
- 确认界面显示行数和风险原因，并提供“粘贴”与“转为单行”
- 不记录粘贴内容，不建立剪贴板历史

### P1-S：链接

- URL 自动识别
- OSC 8 超链接
- Command 点击打开，悬停提供链接反馈，右键可复制链接

链接元数据必须使用逻辑行旁路范围表，仅在实际出现链接时分配。禁止给每个 cell
增加链接字段或对象引用；reflow、scrollback 环覆盖和搜索定位后仍要保持范围正确。

### P1-A：多项目工作流

- 标签设置最小、理想和最大宽度，空间不足时进入溢出菜单，活动标签始终可见
- 会话布局恢复按上文边界实现
- Finder 文件夹可拖入侧边栏添加为项目；侧边栏内部拖动继续只负责排序
- 后台状态只在非活动标签或应用失焦时提示，选中标签后清除
- 系统通知只用于应用不活跃时的明确完成 / 失败事件，不为普通输出制造通知噪声

### P1-A：原生操作入口

- 终端右键菜单：复制、粘贴、打开链接、查找、分屏、清除 scrollback
- 鼠标上报开启时保留 TUI 语义，并提供显式修饰键手势打开 Ink 原生菜单
- 字号放大、缩小、恢复默认值：`⌘+`、`⌘-`、`⌘0`
- 菜单与快捷键支持聚焦相邻 pane
- 明确区分“关闭当前 pane”和“关闭整个标签”，危险关闭沿用会话关闭确认策略

### P1-B：后续增强

- 搜索增加大小写切换、仅搜索选区，以及复制匹配所在命令输出
- 快捷键自定义（已实现原生单次组合键与专用分屏前缀；不包含宏或通用 chord）
- OSC 52 剪贴板（SSH 中复制回本地）
- 通知（OSC 9 / 777）

### 已落地的 P1 基础

- scrollback 搜索：增量、高亮
- 配置文件（TOML）+ 热重载
- 主窗口内嵌设置中心：外观、启动侧边栏、窗口、字体、光标、交互与 scrollback
- iCloud 配置同步：本机设置变化后自动上传，云端配置由用户手动拉取
- 主题配色、字体配置
- 默认跟随 macOS 浅色 / 深色外观，系统外观切换时实时更新
- 帧率跟随显示器刷新率，按内容脏状态绘制

## 暂缓

| 功能 | 当前不做的原因 | 重新评估条件 |
|---|---|---|
| 全局命令历史数据库 | 涉及隐私、磁盘增长和不同 shell 的语义一致性 | 用户明确需要跨会话检索，且能给出数据保留边界 |
| 持久化完整 scrollback | 与快速启动、低内存和敏感输出保护冲突 | 有明确恢复场景，并能提供默认关闭、容量上限和加密策略 |
| 命令面板 | 当前操作数量可由原生菜单和快捷键覆盖 | 菜单层级明显失控或高频操作无法稳定分配快捷键 |
| 设置页分类导航 / 搜索 | 当前单页分组仍能容纳设置项 | 设置项增加后需要频繁滚动或用户找不到入口 |
| 内置文件浏览器 / Git 面板 | 会把终端外壳扩成轻量 IDE | 产品定位明确改变，并单独讨论维护与内存成本 |

---

## 明确不做

| 功能 | 理由 |
|---|---|
| Sixel / Kitty 图片协议 | 实现量大，与"内存小"定位冲突，受众窄 |
| tmux control mode | 集成复杂度极高，iTerm2 投入多年 |
| 内置 SSH / 端口转发 | 超出终端职责 |
| 插件系统 / 脚本引擎 | 引入运行时即引入常驻内存开销 |
| Linux / Windows 移植 | 与 Swift 选型冲突，要做需推翻技术栈 |
| AI 集成 | 除非确立为产品差异点，否则是范围蔓延 |

## 里程碑

| | 目标 | 内容 |
|---|---|---|
| **M1** | 能跑 | PTY 打通，`NSTextView` 占位显示输出，验证 shell 交互 |
| **M2** | 自绘 | Metal 渲染管线 + glyph atlas 替换占位视图 |
| **M3** | 正确 | VT 兼容性、字符宽度、scrollback、选中复制、OSC 133 标记 |
| **M4** | 输入法 | `NSTextInputClient` 完整实现，中文输入可用 |
| **M5** | 外壳 | 侧边栏、标签页、配置文件 |
| **M6** | 达标 | 内存与帧率实测，scrollback 压缩落地，reflow |
| **M7** | 可信 | 安全粘贴、URL / OSC 8、原生终端菜单与字号快捷键 |
| **M8** | 语义 | OSC 133 命令跳转、复制命令 / 输出、耗时、退出状态与标签反馈 |
| **M9** | 连续 | 会话布局恢复、标签溢出、Finder 拖入项目与多 pane 键盘操作 |

M1 先用占位视图而非直接上 Metal，是为了早期就有可运行、可感知的东西，避免长时间陷在渲染管线里。

指标验收（见 [tech-stack.md](tech-stack.md)）：空闲单窗口 < 50MB，10 万行 scrollback < 200MB，全屏重绘稳定 120fps。

M7–M9 的每项能力都必须能独立合并和回滚，不得让某个里程碑依赖下一里程碑才能
使用。推荐交付顺序：安全粘贴 → OSC 133 跳转与复制 → 标签溢出与字号快捷键 →
URL / OSC 8 与右键菜单 → 会话布局恢复 → 命令状态与通知。
