# 标签宽度与溢出菜单设计

Issue：[#64](https://github.com/CheneyCh0u/ink/issues/64)

## 背景

Ink 的顶部标签栏目前使用 `NSStackView.fillEqually`，所有标签始终平分侧边栏开关与
新建标签按钮之间的空间。标签数量增加或窗口变窄时，每个标签会被持续压缩，标题、
关闭按钮和快捷键提示最终挤在一起。

Roadmap 的 P1-A 已明确要求为标签设置最小、理想和最大宽度，空间不足时进入溢出
菜单，并保证活动标签始终可见。本次只完成这一项，不改变标签、项目或会话的数据模型。

## 范围

- 标签最小宽度为 112 pt，理想宽度为 168 pt，最大宽度为 240 pt。
- 空间充足时显示全部标签；空间不足时只显示一个连续区间。
- 活动标签始终位于可见区间内。
- 可见区间只做保证活动标签可见所需的最小移动，避免每次切换都重排整条标签栏。
- 隐藏标签进入位于新建标签按钮左侧的原生 AppKit 溢出菜单。
- 菜单按标签原始顺序显示，选择后使用原始索引执行现有标签切换流程。
- 窗口缩放、侧边栏三态和设置页显示期间都沿用同一套布局规则。

本次不实现横向滚动、最近使用排序、标签拖动重排、菜单内关闭或重命名、标签宽度
设置项、跨项目标签菜单、会话恢复和后台状态点。

## 方案选择

采用“连续可见区间 + 原生溢出菜单”。活动标签离开当前区间时，将区间向前或向后
移动到刚好包含活动标签；只要活动标签仍在当前区间，普通切换、标题变化和同容量刷新
都不移动区间。

没有采用“固定显示最前面的标签并把活动标签钉在末尾”。该方案会产生不连续顺序，
用户无法从顶部栏判断两个可见标签之间隐藏了多少内容。也没有采用横向滚动，因为它
不符合 roadmap 指定的溢出菜单，且会引入额外滚动位置和手势状态。

溢出入口使用 `NSButton` 打开 `NSMenu`。macOS 14 已提供
`NSMenu.popUp(positioning:at:in:)`；不使用 macOS 15 才增加的 `NSPopUpButton`
便利构造器，也不引入第三方控件。

## 宽度语义

三个宽度不是三个窗口断点，而是单个标签的布局约束：

- 112 pt 是硬压缩下限。可见标签不得小于该宽度。
- 168 pt 是短标题的首选宽度。
- 240 pt 是长标题的扩展上限，超过后继续沿用现有头部截断并保留路径尾部。

每个标签先根据标题、关闭按钮、快捷键提示和内部间距得到内容首选宽度，再夹在
168...240 pt。若全部首选宽度能放下，标签按各自首选宽度显示，剩余区域保留为可拖动
标题栏。若首选宽度总和放不下，但所有标签以 112 pt 仍能放下，则按相同压缩比例缩短，
任何标签都不低于 112 pt。

只有当全部标签的最小宽度与间距仍超出可用空间时才进入溢出模式。进入溢出模式后先
为 28 pt 的溢出按钮及其间距留位，再计算最多能容纳的可见标签数量。窗口的 520 pt
最小宽度保证扣除侧边栏开关、新建与设置按钮后仍能容纳至少一个最小标签和溢出按钮。

这些值加入 `InkDesignTokens.TabBar`，业务视图不散落宽度常量。溢出按钮和现有设置
按钮共用 28 pt 点击区域、语义颜色、圆角和 hover 反馈。

## 纯布局模型

新增不持有 AppKit 视图的 `TabBarLayout`。输入包括：

- 标签的首选宽度数组；
- 活动标签原始索引；
- 标签区可用宽度；
- 上一次可见索引区间。

输出包括：

- 连续的可见索引区间；
- 按原始顺序排列的隐藏索引；
- 每个可见标签的最终宽度；
- 是否显示溢出按钮。

布局规则如下：

1. 标签为空时返回空区间，不显示溢出按钮。
2. 全部最小宽度能放下时显示所有标签，并清除旧的局部区间。
3. 需要溢出时按最小宽度计算容量，容量至少为 1。
4. 先把旧区间起点夹到新容量的合法范围。
5. 若活动索引在区间左侧，把起点移到活动索引。
6. 若活动索引在区间右侧，把起点移到 `active - capacity + 1`。
7. 若活动索引已在区间内，保持起点不变；窗口放大时优先向右增加可见标签，接近末尾
   时再向左补齐。
8. 最终宽度先使用首选值，空间不足时公平压缩到最小值，不依赖标题顺序。

调用者传入的活动索引无效时，布局模型按第一个标签处理，避免视图崩溃；正常产品路径
仍由 `Project.activeTabIndex` 保证有效。布局模型只做 O(n) 数组计算，不创建菜单、视图
或字符串。

## 视图结构与数据流

`TabBarView` 继续接收完整的 `[Tab]`，回调继续使用原始数组索引，不改变
`MainWindowController` 和 `Project` 的标签接口。

`TabBarView.reload` 保存完整模型并重建标签项，然后根据当前边界调用布局模型。
`layout()` 只在边界改变或模型刷新时重新计算：

1. 测量每个标签的首选宽度。
2. 计算标签区可用宽度。
3. 调用 `TabBarLayout`，传入上一次可见区间。
4. 只把可见索引对应的 `TabItemView` 放入横向栈，并应用输出宽度。
5. 有隐藏索引时显示溢出按钮并重建菜单；没有时隐藏按钮并释放菜单项。

溢出按钮位于标签栈与新建标签按钮之间，使用 `chevron.down` SF Symbol，tooltip 和
辅助功能标签均为“更多标签”。菜单只列隐藏标签，不复制关闭或重命名入口。每个
`NSMenuItem.tag` 保存原始索引；菜单动作读取 tag 后调用 `onSelect`。即使标题重复，
也不会依赖标题反查索引。

从溢出菜单选择标签后，`MainWindowController.selectTab(at:)` 继续负责退出设置页、
更新活动标签、重建工作区和刷新顶部栏。刷新后的布局收到新活动索引，会最小移动可见
区间并把所选标签显示在顶部。`⌘1` 到 `⌘9`、前后标签命令和新建标签沿用同一路径，
因此也自动保证活动标签可见。

关闭可见标签后，现有 `Project.removeTab` 先规范活动索引，再刷新顶部栏。布局模型会把
旧区间夹回合法范围，避免删除末尾标签后留下空位。重命名只改变首选宽度；若最小宽度
仍能容纳全部标签，不会无故进入溢出模式。

## 交互与辅助功能

- 溢出按钮仅在存在隐藏标签时出现，不保留空白占位。
- 菜单项沿用完整标签标题；前九个标签同时显示现有快捷键提示。
- 菜单顺序始终与项目标签数组一致，不能按可见与隐藏重新排序。
- 菜单项使用原生键盘导航、VoiceOver 朗读和高对比度表现。
- 活动标签不会进入溢出菜单，因此顶部栏始终提供活动状态的非菜单视觉反馈。
- 标签项原有单击、悬停关闭、双击重命名和路径头部截断保持不变。
- 标签栏剩余空白继续允许拖动窗口；标签、溢出、新建和设置按钮不触发窗口拖动。

## 边界与失败处理

布局模型对空数组、非有限或负可用宽度、无效活动索引和失效旧区间给出确定结果，不把
异常输入传给 Auto Layout。产品窗口最小宽度保证正常路径能显示至少一个标签；测试仍
覆盖容量为 1 的情况。

菜单动作收到越界 tag 时直接忽略。标签在菜单打开期间被关闭或切换的可能性由主线程
串行事件处理限制，但动作仍以当前完整模型校验索引，避免使用过期项访问数组。

布局只属于 `InkShell`，不修改 `TerminalCore`、PTY、Metal、配置和持久化格式。它不在
输出回调或逐帧渲染中运行，也不增加 per-cell、per-line 或每个后台终端的常驻数据。

## 测试与验收

纯布局测试覆盖：

- 空标签、单标签和全部标签可见；
- 首选宽度在 168...240 pt 内夹取，压缩后不低于 112 pt；
- 刚好达到与刚好越过溢出阈值；
- 容量为 1，活动标签位于首部、中部和尾部；
- 活动标签留在当前区间时区间不移动；
- 活动标签越过左右边界时只移动到刚好可见；
- 窗口缩小、放大以及删除末尾标签时旧区间合法化；
- 无效活动索引、负宽度和非有限宽度安全归一化。

视图测试覆盖：

- 空间充足时不显示溢出按钮；
- 空间不足时可见标签 frame 不重叠，活动标签存在且宽度不低于最小值；
- 溢出菜单只包含隐藏标签，标题、顺序和 tag 使用原始索引；
- 点击重复标题的菜单项仍选择正确标签；
- 窗口边界变化后菜单与可见区间同步更新；
- 现有侧边栏、新建、设置、标签选择、关闭和重命名回调不回归。

集成与人工验收：

1. 运行 `swift test` 与 `swift build`，要求全部通过且无警告。
2. 打包并启动真实 Ink App，创建至少 12 个标题长短不同的标签。
3. 在 520 pt 最小窗口宽度与常用宽度之间连续缩放，确认按钮不重叠、活动标签不消失。
4. 从溢出菜单选择首、中、尾部隐藏标签，确认顶部栏只做最小移动且终端内容正确切换。
5. 验证 `⌘1` 到 `⌘9`、前后标签、新建、关闭、重命名、设置页和侧边栏三态。
6. 在浅色、深色与 VoiceOver 下确认溢出按钮和菜单可识别。

本改动不触碰终端渲染和 grid 热路径，不要求新增 Instruments 性能采样。完成后仍需确认
窗口持续缩放时布局没有明显卡顿；若出现卡顿，以布局调用次数和标签数量为基线定位，
不能把计算移入逐帧渲染路径。

## 发布与回滚

本功能通过 Issue 分支和关闭 #64 的 PR 合入，不创建版本标签。改动不含数据迁移、配置
键或持久化状态，回滚只需撤销标签栏布局与菜单代码，不影响已有项目和会话数据。
