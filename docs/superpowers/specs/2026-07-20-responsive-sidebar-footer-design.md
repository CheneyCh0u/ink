# 侧边栏底部响应式布局设计

## 目标

侧边栏底部只保留“新建项目”和“设置”两个入口。展开态把它们放在同一行，
图标态继续上下排列，减少展开态的垂直占用，同时保证窄侧边栏中的点击面积。

## 布局

- 分隔线固定在整个底部操作区上方，与项目列表分区。
- 展开态：两个按钮同高、等宽、横向排列，左侧“新建项目”，右侧“设置”。
- 图标态：两个按钮保持现有高度，纵向排列，只显示并居中图标。
- 两种状态都隐藏快捷键提示；快捷键本身不变。
- 复用现有 hover、设置选中背景、圆角和颜色，不新增视觉 token。

## 约束切换

`SidebarViewController` 保存展开态和图标态两组 footer 约束，
`updateDisplayMode()` 在模式变化时只激活对应的一组，避免同时存在的横排、竖排约束冲突。

## 验证

AppKit 布局测试分别验证展开态的同一水平中心线、左右顺序、等宽，
以及图标态的同一水平中心点、上下顺序。完整运行 `swift test` 与 `swift build`。
