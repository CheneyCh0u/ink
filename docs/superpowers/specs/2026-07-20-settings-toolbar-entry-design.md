# 设置入口顶部化设计

## 背景与目标

当前侧边栏底部把“新建项目”和“设置”并列为同等级操作。新建项目属于项目导航的
主要操作，设置则是低频的应用级入口。二者并列会放大设置的视觉权重，也让侧边栏
同时承担项目导航和应用导航两种职责。

本次重构保留侧边栏底部的“新建项目”，将设置改为顶部标签栏最右侧的齿轮图标。
顶部栏在终端与设置状态之间保持可见，让齿轮具备稳定位置和明确选中状态，同时允许
用户直接点击任意标签返回终端。

本次不改变设置项内容、配置文件格式、快捷键，也不涉及 `TerminalCore`、Metal 渲染
或 grid 热路径。

## 方案比较

设计阶段比较了三个位置：

1. 侧边栏底部右下角。设置与新建项目仍在同一区域，层级清晰，但设置仍依赖侧边栏
   可见性。
2. “项目”标题右侧。节省空间，但容易被理解为项目级设置。
3. 顶部标签栏最右侧。设置成为稳定的应用级入口，侧边栏只负责项目，代价是需要让
   顶部栏在设置页中继续保留。

采用方案 3。它在展开、图标和隐藏三种侧边栏状态下都有相同位置，应用级语义也最
明确。

## 视觉层级

顶部栏从左到右依次包含：

- 现有标签；
- 现有新建标签按钮；
- 可伸缩空白区；
- 固定在最右侧的设置齿轮。

齿轮使用 SF Symbol `gearshape`，复用现有标签栏的文字颜色、hover 背景、选中背景、
圆角与动效 token，不新增装饰性颜色、阴影或渐变。默认状态保持低对比度，hover 时
显示背景，设置页打开时使用与当前标签一致的安静选中底色。

齿轮的图标尺寸与现有顶部操作保持一致，点击区域不得小于现有新建标签按钮。按钮
提供“设置”辅助功能标签和包含 `⌘,` 的 tooltip。颜色不是唯一状态提示，选中态同时
通过固定位置和背景形状表达。

侧边栏底部继续保留上方发丝分隔线：

- 展开态：“新建项目”保留整行点击区域，加号与文字左对齐项目内容列；默认无背景，
  hover 时显示现有浅色圆角背景；内容与背景左边缘之间保留 `Spacing.xs` 内边距；
- 图标态：只显示居中的加号，保留 tooltip 与辅助功能标签；
- 隐藏态：顶部齿轮不受影响。

## 视图结构

当前 `tabBar` 位于 `terminalWorkspace` 内，打开设置时整个
`terminalWorkspace` 被隐藏。重构后，`contentRoot` 按垂直方向承载三部分：

1. 始终可见的 `tabBar`；
2. 始终可见的发丝分隔线；
3. 占据剩余空间的内容容器。

内容容器内部叠放终端工作区和设置页，二者继续通过 `isHidden` 切换。终端工作区只
承载 `WorkspaceViewController` 的视图，不再承载顶部栏。这样设置切换不会重建标签、
终端 pane 或设置控制器，也不会改变主窗口 frame。

`TabBarView` 增加设置按钮以及两个明确接口：

- `onSettings`：用户点击齿轮时通知窗口控制器；
- `setSettingsSelected(_:)`：由窗口控制器同步设置页是否可见。

设置切换仍由 `MainWindowController` 统一管理。`TabBarView` 不直接依赖配置或设置页，
只负责呈现和发送用户意图。

## 交互与状态切换

### 从终端进入设置

点击齿轮、选择应用菜单中的设置，或按 `⌘,`，都执行现有 `showSettings(_:)`：

1. 取消待定分屏快捷键并关闭 pane 搜索；
2. 安装或更新设置页；
3. 隐藏终端内容，显示设置内容；
4. 将顶部齿轮设为选中态；
5. 将第一响应者移入设置页。

标签栏在全过程中保持可见。设置已经显示时再次点击选中的齿轮，退出设置并返回当前
终端。

### 从设置返回终端

以下操作都可离开设置：

- 按设置页现有的 Escape 返回动作；
- 选择侧边栏中的任意项目；
- 点击顶部任意标签。
- 再次点击选中的设置齿轮。

点击标签时，先退出设置，再执行原有标签选择。若点击的是当前标签，也要退出设置并
把焦点还给当前 pane。退出后清除齿轮选中态，恢复终端内容并聚焦活动 pane。

项目选择继续沿用当前行为。侧边栏完全隐藏时，齿轮、菜单和 `⌘,` 都仍可用。

设置项保持当前的即时保存与即时应用语义，不显示“完成”或“保存”按钮，避免暗示用户
还需要提交改动。

### 标签与设置状态

设置不是一个标签，不进入标签数组，不参与关闭、重命名、拖动或会话恢复。设置页
显示期间保留原活动标签的视觉状态，齿轮额外显示选中背景，表达“当前标签仍是返回
目标，设置是覆盖在工作区上的全局页面”。

设置页显示期间，新建标签按钮保持可用。用户创建新标签时先退出设置，再沿用现有
逻辑创建并聚焦新会话。

## 错误处理与可访问性

配置保存、打开配置文件和重置失败继续使用现有错误处理，不因入口位置变化而改变。
若设置视图尚未安装，首次打开仍采用惰性安装；安装完成前不改变窗口尺寸。

齿轮必须具备：

- `accessibilityLabel` 为“设置”；
- tooltip 为“设置（⌘,）”；
- 键盘菜单入口 `⌘,`；
- 与现有按钮一致的 hover、按压和焦点反馈；
- 跟随浅色、深色、Reduce Motion 与 Reduce Transparency。

## 性能与内存

此次只调整 `InkShell` 的常驻外壳视图层级，并新增一个系统图标按钮，不增加 per-cell
或 per-line 存储，也不进入渲染循环。设置页仍惰性安装，终端工作区与设置页只切换
可见性，不销毁或重建 PTY、pane 和 Metal 视图。

## 文档同步

实现时更新 `docs/design-system.md` 的“内嵌设置页”章节，删除侧边栏底部设置入口的
旧描述，记录顶部齿轮、共同顶部栏和返回行为。`docs/roadmap.md` 已包含主窗口内嵌
设置中心，本次不改变功能范围，无需调整路线图。

## 验证

### 自动化测试

- `ProjectSidebarTests`：展开态只有一个全宽“新建项目”按钮；图标态只有一个居中
  加号；两种状态都保留分隔线、tooltip 和辅助功能标签。
- `TabBarView` 布局测试：齿轮位于最右侧，不与标签或新建标签按钮重叠；设置选中态
  可切换；不同侧边栏模式下位置稳定。
- `SettingsWindowTests`：打开设置后顶部栏仍可见，窗口 frame 不变；设置页不显示
  “完成”按钮；点击当前标签、其它标签和选中齿轮都能返回终端；齿轮选中态与设置页
  可见性同步。
- 完整运行 `swift test` 与 `swift build`。

### 真实界面检查

从 Issue 分支运行 Ink，并在浅色和深色外观下检查：

- 展开、图标、隐藏三种侧边栏状态；
- 无标签、单标签、多个长标题标签；
- 终端态、齿轮 hover、设置选中态；
- 点击当前标签、其它标签、选中齿轮和 `⌘,` 的焦点返回；
- 设置页首次打开与重复打开时窗口不缩放、不跳动；
- VoiceOver 能读出“设置”，tooltip 能说明 `⌘,` 快捷键。
