# 侧边栏项目行长路径设计

## 背景

项目侧边栏最初把完整缩写路径作为主标题。第一轮修复只保留重建前的悬停状态，未固定
文字区域；第二轮把项目名与父路径拆成两行并保留关闭按钮宽度，但最外层水平栈仍按
内容固有宽度排列。结果是关闭按钮跟随每行文字长度，新增项目后的首次 hover 会在不同
横坐标突然出现按钮，列表也无法形成整齐的右侧操作列。

本改动仍属于路线图已有的 P1 项目侧边栏，不改变功能范围。

## 目标

- 主标题突出最终目录名，而不是展示整条路径。
- 父路径超过可用宽度时从头部省略，保留靠近项目的目录层级。
- 标签状态始终可见，不被父路径截断吞掉。
- 关闭按钮出现、消失和项目行重建均不改变文字区域宽度。
- 项目备注、完整路径、紧凑态和现有交互保持可用。

## 选定方案

采用“项目名优先”的两行布局：

- 第一行显示最终目录名，例如 `wise-studio`。
- 第二行左侧显示父路径，例如 `…/work/code/wiselaw`；有项目备注时用备注替代父路径。
- 第二行右侧固定显示 `未打开` 或 `N 个标签`。
- 最右侧为固定宽度的关闭按钮槽位。

未采用单行完整路径，因为信息密度高，省略后识别仍慢；也不采用面包屑符号，因为层级
符号会增加视觉噪音。最终目录名是项目列表中最有辨识度的信息，父路径只承担消歧。

## 数据与展示模型

`Project` 继续以完整目录 URL 作为事实来源，持久化格式不变。生成侧边栏行时拆出以下
展示字段：

- `title`：最终目录名；用户主目录继续显示 `~`。
- `parentPath`：使用 `~` 缩写后的父目录路径。
- `note`：可选备注；存在时优先于 `parentPath`。
- `status`：`未打开` 或标签数量。

`SidebarViewController.Row` 接收这些展示字段，不在视图内部解析文件路径。完整缩写路径
继续作为项目行 tooltip 和辅助功能描述，避免视觉省略造成信息不可达。

## 布局与交互

展开态项目行保留现有颜色圆点和文件夹/置顶图标。文字列改为两行：主标题单行尾部
省略；副标题为水平布局，左侧备注或父路径可压缩，父路径采用头部省略，右侧状态不
压缩。

项目行不再依赖水平 `NSStackView` 按内容固有宽度分配最外层列。颜色圆点、项目图标、
文字列和关闭按钮分别使用明确锚点：关闭按钮槽位固定在项目行最右侧，文字列固定在
项目图标与关闭按钮槽位之间并占满全部剩余宽度。不同项目名称和父路径的长度不会改变
关闭按钮的横坐标。

关闭按钮槽位使用设计 token 固定宽度，并始终参与布局。悬停状态只切换按钮透明度、
启用状态和辅助功能可见性，不切换 `isHidden`。`hoveredRowPath` 继续跨 `reload()` 保留
悬停项，避免点击重建时关闭图标自身闪烁。新增项目后即使指针已经位于新项目行范围内，
首次 hover 也只能显示固定在右侧的按钮，不得改变任何文字或按钮 frame。

紧凑态仍只显示颜色竖条与项目图标，不创建关闭按钮或文字列。拖拽、右键菜单、置顶、
颜色标记和点击选择逻辑不变。

## 测试与验收

- 路径展示测试覆盖普通目录、用户主目录和父路径缩写。
- AppKit 布局测试使用超长路径，记录转换到项目行坐标系后的标题、父路径和状态 frame。
- hover、选择触发的 reload、mouse exit 三个阶段中，上述 frame 必须完全一致。
- 关闭按钮槽位宽度固定；透明状态仍参与布局，只有悬停时可点击。
- 使用标题和父路径长度不同的两行，断言两个关闭按钮转换到各自行坐标后的 `minX`
  完全一致，并与项目行右边缘保持固定距离。
- 从一行 reload 为两行后，对新增行执行第一次 `mouseEntered`，断言新增行标题、父路径、
  状态和关闭按钮 frame 均不变化。
- 父路径实际宽度小于其完整固有宽度，且 line break mode 为头部省略；状态完整显示。
- 紧凑态不创建项目关闭按钮。
- 运行 `swift test`、`swift build`，并在当前分支构建的真实 Ink 进程中用长路径来回点击
  验证。

改动只涉及 AppKit 外壳与设计 token，不进入 `TerminalCore`、Metal 渲染或 grid 热路径，
无需 Instruments 性能采样。
